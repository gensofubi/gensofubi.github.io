Preliminary Results

Message Passing Model

We achieve parallelism by having each process compute a portion of the data and then use message communication protocols to combine the results. The speedup effect is as follows:

Process num 1 2 4 8
Total speed-up 1x 1.92x 3.67x 6.07x
Data loading speed-up 1x 1.98x 3.75x 6.43x
Max stack speed-up 1x 1.12x 1.33x 1.50x

From this, we can observe that the speedup for the data loading part is nearly perfect, but the speedup for the max stack part is quite small. This is because the message communication involves transferring the entire pixel data of the image, and the large size leads to significant communication overhead. In the future, we can further investigate better ways to transmit the data.

Data Parallel Model

We achieve parallelism by having each process taking a portion of the data-loading and max-stack-computing tasks. The speedup effect is as follows:

Process num 1 2 4 8
Total speed-up 1x 1.72x 2.60x 4.75x
Data loading speed-up 1x 1.73x 2.70x 5.19x
Max stack speed-up 1x 1.48x 1.55x 1.76x

From this, we can observe that the speedup for the data loading part is good, but the speedup for the max stack part is still quite small. This is because the accessing and updating of the Max Stack result is critical and limited the parallelism of the program.
